# Stakeholder Updates & Engineering Visibility

> **You are here**: On the Job — SDE2 → Principal
> **Roadmap**: [Developer Master Roadmap](../ROADMAP.md) | **Prerequisites**: [RFC/ADR](04_RFC_ADR_Writing.md), [Executive Communication](../00_Interview_Prep/Principal/Executive_Communication_Board_Narrative.md) | **Next**: [Product Partnership](../00_Interview_Prep/Career/Product_Partnership.md)

Invisible good work does not get promoted. Loud bad work gets you fired. This guide is how to stay **visible with integrity** — updates, demos, and written artifacts that busy stakeholders actually read.

---

## Who needs what

| Audience | Cadence | Length | Cares about |
|----------|---------|--------|-------------|
| EM | Weekly | 5 bullets | Delivery risk, people |
| PM | Weekly / sprint | Outcome + scope | Metric, date, cuts |
| Peer teams | Per RFC / milestone | Interface changes | Contracts, timelines |
| Director / VP | Biweekly / monthly | One-pager | Risk, ask, ROI |
| Skip-level | Monthly | 3 bullets + career | Truth from the floor |

---

## The 5-bullet weekly update

```
1. Outcome: [metric or milestone] — green / yellow / red
2. Shipped: [what users/systems gained]
3. Next: [what lands in 7 days]
4. Risk: [one sentence] → Need: [decision or help]
5. Debt / reliability: [one line if relevant]
```

**Yellow/red without a need** is venting. Always pair risk with an ask.

---

## Demo discipline

| Do | Don't |
|----|-------|
| Show the user path or ops dashboard | Only slides of architecture |
| Call out what is stubbed | Fake success on happy path only |
| Time-box 10–15 min + questions | 45-min monologue |
| Record for async (GCC / remote) | Require every stakeholder live |

## Slack / email hygiene

| Pattern | Example |
|---------|---------|
| Subject/intent first | "[Decision needed by Thu] Payment flag default" |
| Options table | A / B / C with recommendation |
| FYI vs Action | Label clearly |
| Async-first (GCC) | Write the update; meeting only for conflict |

Exec translation of metrics: [Executive Communication](../00_Interview_Prep/Principal/Executive_Communication_Board_Narrative.md).

## Visibility without politics

| Healthy | Unhealthy |
|---------|-----------|
| Share credit by name | Steal credit in skip-level |
| Surface risks early | Hide until launch week |
| Demo unfinished with caveats | Only appear for wins |
| Document decisions | Private agreements that surprise the team |

---

## Interview angle

"Tell me about a time you influenced without authority" → use a written RFC + stakeholder update trail, not "I messaged people a lot."
